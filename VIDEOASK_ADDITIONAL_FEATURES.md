## 1. チーム機能・権限管理

### 1.1 ロール（役割）

VideoAskには**2つの主要ロール**があります：

#### Admin（管理者）
- すべての機能にアクセス可能
- チームメンバーの招待・管理
- すべてのフォルダとVideoAskへのアクセス
- 組織設定の変更
- 請求管理

#### Member（メンバー）
- 制限付きアクセス
- 特定フォルダのみアクセス可能（管理者が設定）
- VideoAskの作成・編集・削除（アクセス権のあるフォルダ内）

### 1.2 フォルダベースのアクセス制御

#### 機能
- **フォルダごとにアクセス権を設定**
- メンバー招待時に特定フォルダへのアクセスをON/OFF可能
- 後からアクセス権の変更も可能

#### 重要な制限事項
**Default Folder（デフォルトフォルダ）**
- すべてのチームメンバーが閲覧・編集・削除可能
- **権限設定に関わらず、全員がアクセスできる**
- 注意: 機密情報は別フォルダに配置すべき

### 1.3 チームメンバー管理

#### 招待フロー
```
1. Account & Org settings → Team をクリック
   ↓
2. メールアドレスを入力
   ↓
3. ロール選択（Admin or Member）
   ↓
4. （Memberの場合）フォルダアクセス権を設定
   ↓
5. 招待メール送信
   ↓
6. 受信者がアカウント作成（持っていない場合）
   ↓
7. 招待を承認
```

#### 権限変更
- いつでもロールやフォルダ権限を変更可能
- メンバー名の横のロールをクリックしてドロップダウンから変更

#### アカウント要件
- **各チームメンバーは独自のVideoAskアカウントが必要**
- アカウントを持っていない場合は招待承認前に作成を促される

## 2. メール通知・SMS統合

### 2.1 メール通知機能

#### 通知設定
- **自動送信**: 新しい回答を受信した際に通知
- **オン/オフ切り替え**: 通知を完全に無効化可能
- **ワークフローに合わせた制限**: 受信数を調整可能

#### 通知先
- アカウントに関連付けられたメールアドレスに送信

#### 詳細設定
1. **すべてのVideoAsk vs 個別のVideoAsk**
   - 組織内のすべてのVideoAskに対する通知
   - または特定のVideoAskのみ

2. **会話ベースの通知**
   - すべての回答で通知を受け取る
   - または自分が会話に参加している場合のみ

#### 回答者への通知
- **自動メール通知**: 作成者が返信するたびに回答者にメール送信
- **無効化可能**: VideoAskごとに無効化し、代替手段（SMS、Zapier経由の別メールサービス）を使用可能

### 2.2 SMS統合

#### 統合方法
- **Zapier経由**
- **Pipedream経由**
- **その他の統合プラットフォーム**（Integrately、Pabbly Connect等）

#### ユースケース
- **新規回答の即時通知**
  - 回答ごとにSMS送信
  - モバイルブラウザで確認できるリンク付き

- **外出先での確認**
  - メールより開封率が高い（97%）
  - 緊急性の高い回答の通知に最適

#### 統合ワークフロー例
```
VideoAsk新規回答（トリガー）
   ↓
Zapier
   ↓
SMS送信（Twilio、SMS Gateway Center等）
   ↓
管理者のスマートフォンに通知
```

## 3. キャプション・字幕・アクセシビリティ

### 3.1 クローズドキャプション（字幕）

#### 機能
- **自動生成**: ビデオの音声から自動的に字幕生成
- **作成者と回答者の両方で利用可能**
- **アクセシビリティ向上**: 聴覚障害者にも対応

#### 回答者側の操作
- ビデオコントロールの「CC」ボタンをクリック
- 字幕のオン/オフ切り替え

#### 作成者側の設定
1. **ビデオコントロールの有効化必須**
   - コントロールがオフの場合、CCボタンも表示されない

2. **デフォルト表示設定**
   - 「Show captions by default」オプション
   - 自動的に有効化（Settings内）
   - ミュート自動再生時から字幕表示

### 3.2 文字起こし言語設定

#### Advanced Settings
- **Transcription Language**: 動画で話されている言語を選択
- 精度向上のため、正しい言語を設定することが重要

#### 対応言語
- 多言語対応（具体的な言語リストは公式ドキュメント参照）

### 3.3 不適切な言葉のフィルタリング

#### Filter Profanities（不適切な言葉フィルタ）
- 不適切な言語を検出
- 文字起こしと字幕から自動的にフィルタリング

### 3.4 多言語VideoAsk（ワークアラウンド）

#### 現状の制限
- **インターフェース言語の変更**: ボタン、プロンプト等のみ
- **コンテンツの翻訳はなし**: オーバーレイテキスト、選択肢等は手動で翻訳必要

#### 多言語対応の方法（回避策）
```
Step 1: 言語選択
  Answer Type: Multiple Choice
  選択肢:
    - English
    - 日本語
    - Español
    - Français
  Logic:
    - English → 英語版VideoAsk（別URL）
    - 日本語 → 日本語版VideoAsk（別URL）
    - Español → スペイン語版VideoAsk（別URL）
    - Français → フランス語版VideoAsk（別URL）
```

**実装方法**
1. 各言語用に別々のVideoAskを作成
2. 最初のステップで言語選択（Multiple Choice）
3. 各選択肢から対応言語のVideoAskにリダイレクト

## 4. コンタクトフォームのカスタマイズ

### 4.1 デフォルトフィールド

#### 標準フィールド（4つ）
1. **Name**（名前）
2. **Email**（メールアドレス）
3. **Phone number**（電話番号）
4. **Product name**（製品名）

### 4.2 フィールドの有効化/無効化

#### 表示設定
- 各フィールドの横の**目のアイコン**をクリック
- 表示/非表示を切り替え

### 4.3 必須/任意の設定

#### Required（必須）設定
- アスタリスク（*）の有効化/無効化
- 必須にすると回答者は入力必須

### 4.4 カスタムフィールドの制限

#### 現状の制限事項
1. **姓と名の分離不可**
   - 現時点では1つの「Name」フィールドのみ
   - 分離機能は機能リクエストボードにあるが、開発ロードマップには未掲載

2. **住所フィールドの複数入力**
   - 住所（street、unit、city、state、zip）を1ページで収集不可
   - 各フィールドを別々のステップで収集する必要がある
   - ユーザーからの不満が多い

#### ワークアラウンド

**方法1: テキストステップを使用**
- 独立したステップを作成
- Answer Type: Open Ended（Textのみ有効化）
- カスタム質問として使用

**方法2: Product Nameフィールドを転用**
- Product Nameフィールドを有効化
- インターフェーステキストを編集して別の質問に変更
- 例: "Product name" → "Company name"、"Job title"等

**方法3: カスタム変数（有料プラン）**
- 有料プランではカスタム変数を追加可能
- より柔軟なデータ収集

### 4.5 Typeform連携による拡張

#### 複雑なフォームの場合
- **Typeformと組み合わせる**ことでより高度なフォーム作成が可能
- VideoAskのコンタクトフォームの制約を回避
- Typeformで詳細情報収集 → VideoAskにデータを渡す

## 5. アナリティクス・トラッキング

### 5.1 UTMトラッキング

#### 概要
- URLパラメータを使用してキャンペーンやトラフィックソースを追跡
- マーケターがキャンペーンパフォーマンスを監視

#### 要件
- **有料プランが必要**

#### 設定方法
1. カスタム変数機能を使用
2. UTMパラメータをVideoAsk URLに追加
3. Google Analyticsに接続
4. アナリティクスレポートで確認

#### UTMパラメータ例
```
https://www.videoask.com/abc123?utm_source=facebook&utm_medium=cpc&utm_campaign=spring_sale
```

### 5.2 Google Analytics統合

#### 対応バージョン
- **Universal Analytics**: 公式サポート
- **Google Analytics 4 (GA4)**: コミュニティガイドによる回避策あり

#### トラッキングイベント
VideoAskから送信されるイベント（前述の通り）：
1. `videoask_presented`
2. `videoask_question_submitted`
3. `videoask_submitted`
4. `generate_lead`

#### 埋め込みページでのトラッキング
- ランディングページに埋め込まれたVideoAskでもUTMタグ使用可能
- トラフィックソースとエンゲージメントを追跡

### 5.3 Facebook Pixel統合

#### 設定方法
1. Facebook Business ManagerからPixel IDをコピー
2. VideoAskの設定にPixel IDを貼り付け

#### 制限事項
- **カスタムコンバージョンイベントは未対応**
- 標準的なエンゲージメント追跡のみ

### 5.4 Google Tag Manager統合

#### 設定方法
- Container IDをVideoAskに挿入
- イベント追跡が可能

### 5.5 高度なUTM機能

#### ランディングページからの受け渡し
- JavaScriptコードを使用
- ランディングページのUTMパラメータを埋め込みVideoAskに受け渡し

**コード例**
```javascript
// ランディングページのUTMパラメータを取得
const urlParams = new URLSearchParams(window.location.search);
const utmSource = urlParams.get('utm_source');
const utmMedium = urlParams.get('utm_medium');
const utmCampaign = urlParams.get('utm_campaign');

// VideoAsk URLに追加
const videoaskUrl = `https://www.videoask.com/abc123?utm_source=${utmSource}&utm_medium=${utmMedium}&utm_campaign=${utmCampaign}`;
```

**確認方法**

## 7. 制限事項・仕様の詳細

### 7.1 ビデオ長さの制限

#### 作成者（VideoAsk作成側）
- **録画**: 最大5分（300秒）
- **アップロード**: 長さ制限なし（ファイルサイズ制限内であれば）

#### 回答者（回答側）
- **デフォルト**: 2分
- **有料プラン**: 最大5分まで延長可能
- **アップロード**: 長さ制限なし（ファイルサイズ制限内）

### 7.2 ファイルサイズ制限

#### 作成者
- **Webブラウザ**: 最大500 MB
- **iOSアプリ**: 最大2 GB

#### 回答者
- **ビデオアップロード**: 200 MB（一部情報源では500 MB）
- **ファイルアップロード答えタイプ**: 最大10 MB（ドキュメント等）

### 7.3 対応フォーマット

#### ビデオ
- .mp4
- .mov
- .webm

#### ファイルアップロード
- ほぼすべてのファイルタイプ（10 MB以内）

### 7.4 推奨仕様（再掲）

- **解像度**: 1920x1080 px（フルHD）
- **アスペクト比**: 16:9（ランドスケープ）
- **自動最適化**: 1920px超は自動で1920pxにリサイズ

## 8. テンプレート機能の詳細

### 8.1 テンプレートの使用

#### テンプレートを使うメリット
- **時間節約**: 同じ構造を何度も作らなくて済む
- **ベストプラクティス**: 業界標準のフローを参考にできる
- **クイックスタート**: すぐに使い始められる

#### テンプレートのコピー
- **処理時間にカウントされない**: テンプレートをコピーするだけでは月間処理時間を消費しない
- **ビデオの差し替え**: テンプレートのビデオを自分のビデオに置き換えると処理時間にカウントされる

### 8.2 カスタムテンプレートの作成

#### ユースケース
- チームで同じ構造のVideoAskを頻繁に作成する場合
- 標準化されたフローを維持したい場合

#### 作成方法
1. VideoAskを作成
2. Settings → Save as template
3. テンプレート名を設定
4. 組織内で共有

#### テンプレートの管理
- テンプレート一覧から編集・削除可能
- チーム全体で利用可能

## 11. 制約事項まとめ

### 11.1 現在の主な制約

1. **コンタクトフォームのカスタマイズが限定的**
   - 姓名の分離不可
   - 複数フィールドの同一ページ表示不可
   - カスタムフィールドが限定的

2. **Facebook Pixelのカスタムコンバージョン未対応**

3. **Google Analytics 4の公式サポートなし**
   - Universal Analyticsのみ公式対応
   - GA4は回避策が必要

4. **ビデオ編集機能なし**
   - トリミング、カット、クロップ等の編集機能なし
   - 外部ツールで編集してからアップロード必要

5. **バックボタンなし**
   - 一方向のフロー
   - 回答者は前のステップに戻れない

6. **送信後の変更不可**
   - 回答送信後は録り直し不可

7. **Default Folderの権限制御不可**
   - デフォルトフォルダは全員がアクセス可能
   - 機密情報の管理に注意必要

## 16. 実装時の推奨事項

### 16.1 ユーザー体験の最適化

1. **モバイル対応を優先**
   - 顔を中央に配置
   - ステップ数を制限
   - 接続不安定性を考慮

2. **ビデオ品質と長さのバランス**
   - 簡潔なビデオ（1-2分推奨）
   - 高画質（1920x1080）
   - ファイルサイズ最適化

3. **アクセシビリティ**
   - 字幕を有効化
   - コントラスト比を確保
   - キーボードナビゲーション対応

### 16.2 データ管理

1. **フォルダ構造**
   - Default Folderには機密情報を置かない
   - プロジェクト/部署ごとにフォルダ分け
   - アクセス権を適切に設定

2. **定期的なバックアップ**
   - 定期的にデータエクスポート
   - Google Sheets連携でリアルタイムバックアップ

3. **GDPR対応**
   - 同意取得を確実に
   - データ削除リクエストに対応できる体制
